¿Que es reposito de Codigo?

Un repositorio de código es un lugar donde el código de una aplicación, de un programa cualquiera está almacenado y desde donde se puede distribuir. Nuestro disco duro podría ser un ejemplo de repositorio, tenemos nuestro código guardado ahí y lo podemos distribuir, y a través de un archivo zip lo podemos en viar mediante cualquier tipo de servicio. Como ya veremos, no es la mejor opción de un repositorio, donde guardar el código de tu programa, de tu software, pero es una alternativa. Estos repositorios están almacenados en servidores, no en ordenadores locales o en un disco duro que tengamos. Esto es muy importante de cara tanto al almacenamiento, a la seguridad, como a la distribución del código como ya veremos ahora. Y la segunda característica que tienen es que utilizan sistemas de control de versiones, para explicarlo de una forma sencilla, es un programa que está manejando todo el código que está guardado en ese repositorio, en esa carpeta. Básicamente lo que hacen es añadir muchas funcionalidades a esa carpeta y mantener un historial de todos los cambios para tener la opción de poder revertirlos, o también nos da la opción de poder trabajar en paralelo varias personas a la vez. No te preocupes si no acabas de entender todos estos conceptos ahora, porque son una explicación muy por encima de todos los beneficios que tienen, porque lo vamos a ver, de una forma rudimentaria, a través de lo que genera guardar el código en un zip en el ordenador. En resumen, un repositorio de código en un servidor, con un sistema de control de versiones, que permite guardar todo nuestro programa, toda nuestra aplicación en un lugar seguro y a través de un control de versiones que permite mantener un historial de todos los cambios que se producen. Dicho esto, y aunque haya sido difícil de entender si nunca has trabajado con un repositorio de código de estos o si nunca has oído hablar de ello, debes ver por qué deberíamos utilizar este tipo de repositorios de código para el software, o dicho de otra manera, ¿por qué no guardar tu código en un archivo zip en tu ordenador o incluso un archivo zip en un servidor ¿por qué no hacerlo así? ¿por qué utilizar este tipo de repositorios?

¿Que es GIT?

Git, es un software de control de versiones diseñado por Linus Torvalds. La pregunta es ¿qué es control de versiones? Pues bien, se define como control de versiones a la gestión de los diversos cambios que se realizan sobre los elementos de algún producto o una configuración del mismo es decir a la gestión de los diversos cambios que se realizan sobre los elementos de algún producto o una configuración, y para los que aún no les queda claro del todo, control de versiones es lo que se hace al momento de estar desarrollando un software o una página web. Exactamente es eso que haces cuando subes y actualizas tu código en la nube, o le añades alguna parte o simplemente le editas cosas que no funcionan como deberían o al menos no como tú esperarías. Y, entonces ¿a que le llamamos sistema de control de versiones? Muy sencillo, son todas las herramientas que nos permiten hacer todas esas modificaciones antes mencionadas en nuestro código y hacen que sea más fácil la administración de las distintas versiones de cada producto desarrollado; es decir Git.

¿Que es GitHub?

GitHub es una compañía sin fines de lucro que ofrece un servicio de hosting de repositorios almacenados en la nube. Esencialmente, hace que sea más fácil para individuos y equipos usar Git como la versión de control y colaboración. La interfaz de GitHub es bastante fácil de usar para el desarrollador novato que quiera aprovechar las ventajas del Git. Sin GitHub, usar un Git generalmente requiere de un poco más de conocimientos de tecnología y uso de una línea de comando. GitHub es tan fácil de usar, que incluso algunas personas usan GitHub para administrar otro tipo de proyectos – como escribir libros. Además de esto, cualquier persona puede inscribirse y ser hospedar un repositorio de código público completamente gratuito, el cual hace que GitHub sea especialmente popular con proyectos de fuente abierta. Como compañía, GitHub hace dinero vendiendo alojamiento para repositorios de código privado, al igual que otros planes enfocados para negocios, que hace más fácil que las organizaciones administren a los miembros de su equipo y su seguridad. Utilizamos GitHub de forma extensiva aquí en Kinsta, para administrar y desarrollar proyectos internos.
